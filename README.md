# Deep Learning Midterm 1 — Text-to-SVG with Qwen 2.5-3B LoRA

## Notebooks

### 1. `starter-notebook.ipynb` — Training & Inference
The main notebook. Run cells top-to-bottom.

- **Cells 1–8**: setup, config, data loading, prompt formatting
- **Cell 9**: loads the model (auto-detects inference vs training mode)
- **Cell 10**: trains the model *(skip if adapter already exists)*
- **Cell 11**: saves the adapter *(skip if already saved)*
- **Cell 12**: runs a single test inference — expected output is a valid SVG string
- **Cell 13**: generates the full submission CSV — skips automatically if a completed CSV already exists

**Output**: `submission_qwen3b_svg_lora_<timestamp>.csv` in the repo root.

---

### 2. `results.ipynb` — Experiment Results
Visualizes training metrics and compares runs.

Run all cells top-to-bottom. No model or GPU needed.

**Output**: inline tables and loss curve plots comparing v1 and v2 runs.

---

### 3. `analyze_run.ipynb` — Submission Failure Analysis
Inspects the debug log from a submission run to categorize generation failures.

Requires a `debug_*.jsonl` file in the repo root (auto-generated by cell 13 of `starter-notebook.ipynb`).

Run all cells top-to-bottom.

**Output**: failure breakdown by category (no SVG tag, truncated, parse error, etc.) and a sample of failed prompts.

---

## Quick Start

```bash
# open each notebook in VS Code
# select the project's venv as the notebook kernel / Python interpreter
# run cells top-to-bottom
```

Recommended order:

1. `starter-notebook.ipynb` — run all cells to train or load the adapter, test generation, and create a submission CSV.
2. `results.ipynb` — run all cells to inspect training metrics and compare runs.
3. `analyze_run.ipynb` — run all cells to inspect a generated `debug_*.jsonl` file and categorize failures.

If you are using the terminal for setup or package installs, activate the same environment there as well.

- macOS/Linux: `source venv/bin/activate`
- Windows PowerShell: `./venv/Scripts/Activate.ps1`


## Notebook State

The committed state of `starter-notebook.ipynb` does **not** contain cell outputs from a full training or inference run. The `artifacts/` folder contains only the saved checkpoints from our best model (`qwen3b_svg_lora`). Cell outputs were cleared before committing to keep the repo lightweight. If you want to see the results of our run, refer to `results.ipynb` and `analyze_run.ipynb`.

## Starting Fresh (Reproducing Our Run)

To avoid the notebook skipping training/inference by detecting our pre-existing outputs, delete the following before running:

```bash
# Saved LoRA adapter (causes Cell 9 to load for inference instead of training)
rm -rf artifacts/qwen3b_svg_lora/

# Any completed submission CSVs (causes Cell 13 to skip inference entirely)
# You could also move them to another location if you prefer that.
rm -f submission_qwen3b_svg_lora_*.csv submission1.csv submission2.csv

# Any in-progress partial runs
rm -f submission_qwen3b_svg_lora_*.csv.part

# Any debug logs from previous runs (optional, won't affect notebook behavior)
rm -f debug_qwen3b_svg_lora_*.jsonl
```

## Setting Up the Jupyter Kernel

To use the project's virtual environment as the notebook kernel, install `ipykernel` and register it:

```bash
source venv/bin/activate
pip install ipykernel
python -m ipykernel install --user --name=venv --display-name "Python (venv)"
```

Then in VS Code, open the notebook and select **"Python (venv)"** from the kernel picker in the top-right.
